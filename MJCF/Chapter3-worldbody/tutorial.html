1 概述

<worldbody> 是 MuJoCo MJCF 模型的根物理节点。一切可仿真的物理实体——地面、机器人、物体、传感器安装点——都必须放置于此标签之内。

本章将详细介绍 <worldbody> 中的三大核心子元素：

G

geom

几何体 · 碰撞体积 · 质量

S

site

简化几何体 · 传感器安装点

B

body

刚体 · 坐标树 · 嵌套层级

2 geom — 几何体

<geom> 是 MuJoCo 中最基础的物理实体。它定义了一个具有形状、质量、碰撞体积和外观的对象。一个 geom 可以是一个球体、一个立方体，也可以是一张从 CAD 导入的网格——MuJoCo 使用它们进行碰撞检测和物理计算。

2.2 size 参数对照表

不同几何体类型的 size 参数含义不同，这是需要特别注意的地方：

typesize 参数个数含义
plane3X 半长; Y 半长; 渲染网格线间距（若半长为 0 则该维度无限延伸）
hfield0忽略，改用高度场文件尺寸
sphere1半径
capsule1 或 2半球半径; [若不用 fromto] 圆柱段半长
ellipsoid3X 半径; Y 半径; Z 半径
cylinder1 或 2半径; [若不用 fromto] 半长
box3X 半长; Y 半长; Z 半长
mesh0忽略，改用网格文件尺寸

💡 提示：除了 size 之外，capsule、cylinder、box 和 ellipsoid 还可以用 fromto 属性来定义几何体——用两个 3D 点指定起点和终点，几何体的 Z 轴正方向为 point2 → point1。

2.3 各类几何体用法示例

🟫 地板（plane）

平面通常用棋盘格纹理来可视化地面。当 X 或 Y 半长为 0 时，平面在该维度无限延伸。

<!-- 棋盘格纹理 -->
<texture name="grid" type="2d" builtin="checker"
         width="2048" height="2048"
         rgb1=".3 .4 .8" rgb2=".9 .9 .9" />
<geom name="ground" type="plane" size="100 100 .01" material="grid" />

⚽ 球体（sphere）

只需一个 radius 参数，最简单的三维几何体。

<geom type="sphere" material="metal_material" size="1" mass="1" />

💊 胶囊体 & 圆柱体（capsule / cylinder）

用两个 size 参数（半径 + 半长），或使用 fromto 直接指定两端点。

<!-- 使用 size 参数 -->
<geom type="capsule" material="metal_material" size="1 2" mass="1" />

<!-- 使用 fromto 参数，Z 轴正方向: point2 → point1 -->
<geom type="cylinder" size="0.1" fromto="0 0 0  0 0 0.2"
      rgba=".5 .5 0 1" />

📦 立方体 & 椭球体（box / ellipsoid）

三个 size 参数：X半长、Y半长、Z半长。

<geom type="box" material="metal_material" size="1 2 1" mass="1" />
<geom type="ellipsoid" material="metal_material" size="1 2 1" mass="1" />

🕸 网格（mesh）

从外部 3D 模型文件（STL/OBJ）导入。先在 <asset> 中声明 mesh，再在 geom 中引用。

<!-- asset 中声明 -->
<mesh name="forearm" file="forearm.stl" />

<!-- geom 中引用 -->
<geom type="mesh" mesh="forearm" material="metal_material" />

2.4 几何体核心属性汇总

  • type — 几何体类型（plane/sphere/box/…）
  • size — 尺寸参数，含义因 type 而异
  • pos — 相对父级 body 的位置 (x y z)
  • fromto — 两点定义法，替代 size+旋转
  • quat / axisangle / euler — 旋转表示（quat 为 wxyz 顺序）
  • material — 引用已定义的材质名
  • rgba — 直接指定颜色 (r g b a)，比材质更省资源
  • mass — 质量 (kg)，与 density 二选一
  • density — 密度 (kg/m³)，与 mass 二选一
  • friction — 滑动/扭转/滚动 三参数摩擦力
  • condim — 接触维度，取值 1/3/4/6
  • priority — 碰撞优先级（数值越大越优先）
  • shellinertia — true=质量集中边缘，false=均匀分布
  • contype — 32位碰撞类型掩码
  • conaffinity — 32位碰撞亲和掩码
3 condim — 碰撞接触维度

condim 决定了两个几何体碰撞时，MuJoCo 会计算哪些摩擦维度——数值越大，模拟越精细，但计算开销也越大。

condim摩擦行为适用场景
1无摩擦接触（仅法向力）纯弹性碰撞、简化模拟
3切向平面摩擦（平移摩擦力）大多数刚体碰撞场景
4切向摩擦 + 绕法线旋转摩擦软接触建模、需要防止绕接触点旋转
6完整摩擦：切向 + 法线旋转 + 切向轴旋转防止物体无限滚动（如球在地板上滚）

📌 物理直观理解：

condim=1 就像冰面上没有摩擦；condim=3 是日常表面摩擦力；condim=6 是带自旋阻力（如橡胶接触），会额外阻止物体的无限滚动。

4 碰撞掩码 — contype & conaffinity

contype

32位掩码，表示"我是什么类型的碰撞体"。当两个 geom 的 contype 和 conaffinity 按位与 (&) 结果非零时，两者就会发生碰撞。

conaffinity

32位掩码，表示"我可以和什么类型的碰撞体发生碰撞"。它决定了一个 geom 对哪些 contype 是"可见的"。

💡 碰撞判定逻辑：

geomA.contype & geomB.conaffinity ≠ 0 且 geomB.contype & geomA.conaffinity ≠ 0

两个方向的按位与都需要非零，碰撞才会发生。这是一种灵活的碰撞分组机制——你可以让某类物体只与特定类型碰撞。

示例：选择性碰撞

下面的配置中，立方体（contype=2, conaffinity=3）只会与 conaffinity 包含 bit 2 且 contype 包含 bit 1 或 bit 0 的对象碰撞：

<geom type="box" size="0.1 0.1 0.1"
      contype="2" conaffinity="3" />
5 site — 简化几何体

📌 一句话理解：site 是 geom 的"轻量版"。它有形状和位置，但不参与碰撞检测，也没有质量。它仅用于可视化标记和传感器安装。

<site> 的典型用途：

  • 在机器人的特定部位安装传感器（力传感器、IMU 等）
  • 标记关节位置或参考坐标系
  • 纯可视化——高亮某些关键点而不影响物理
  • 作为 actuator 的作用点

示例：在球体上安装一个 site

<body pos="-1 0 .5">
    <freejoint />
    <geom type="sphere" size="0.1" rgba=".5 0 0 1" />
    <site type="capsule" size="0.05 0.1" rgba="0 0 .5 1" />
</body>

site 的属性与 geom 高度相似：type、size、pos、rgba 等均可用，但不支持 mass、density、friction 等物理相关属性。site 只能使用简易几何体类型。

6 body — 刚体与坐标树

<body> 是 MuJoCo 中组织物理结构的容器。一个 body 内部可以包含多个 geom、site、joint 以及嵌套的子 body。整个机器人模型就是一棵由 body 嵌套而成的树状结构——这和 ROS 中的 TF 坐标变换树概念几乎一致。

6.1 坐标变换规则

每个 body 的位置 pos 和姿态 quat/euler/... 都是相对于其父级 body 的坐标系。

  • 顶层 body（直接放在 worldbody 下）相对于世界坐标系
  • body 内的 geom 相对于该 body 的局部坐标系
  • body 内的 site 也相对于该 body 的局部坐标系
  • body 内的子 body 相对于父 body 的局部坐标系

⚠️ 关键理解：变换是逐层叠加的。修改父级 body 的位置会带动其下所有子 body、geom 和 site 一起移动。这就是"骨骼带动肌肉"的物理直觉。

6.2 树状结构演示

以下代码展示了三层嵌套的 body 树：

<body name="A" pos="0 0 2" euler="0.5 0 0">
    <freejoint />
    <geom type="sphere" size="0.1" rgba=".2 0 0 1" />
    <!-- 子 body B 相对于 A 向上偏移 0.5 -->
    <body name="B" pos="0 0 0.5" euler="0.5 0 0">
        <geom type="capsule" size="0.05 0.1" rgba="0 0 .2 1" />
    </body>
</body>

6.3 坐标树可视化

以下树形图展示了 scence.xml 中 body A → B 的嵌套关系：

World (世界坐标系) ├── body A pos=(0,0,2) euler=(0.5,0,0) │ ├── geom: sphere size=0.1 ← 相对于 A 的局部坐标 │ └── body B pos=(0,0,0.5) euler=(0.5,0,0) ← 相对于 A │ └── geom: capsule size=0.05 0.1 ← 相对于 B ├── body (pos=-1,0,0.5) ← freejoint ├── body (pos=-0.5,0,0.5) ← freejoint ├── body (pos=0,0,0.5) ← freejoint ├── body (pos=0.5,0,0.5) ← freejoint ├── body (pos=1,0,0.5) ← freejoint ├── body (pos=1.5,0,0.5) ← freejoint └── body (pos=2,0,0.5) ← freejoint, mesh=tetrahedron

💡 B 的世界坐标如何计算？

B 的 pos 是 (0, 0, 0.5)，含义是"在 A 的局部坐标系中向上偏移 0.5"。若 A 在世界坐标 (0, 0, 2) 处且绕 X 轴旋转 0.5rad，那么 B 的世界位置 = A 的位姿矩阵 × B 的相对偏移。这和 ROS TF 树中的 frame_id 父子关系完全一致。

7 旋转表示方式

MuJoCo 支持多种旋转表示方法，可根据需要灵活选择：

属性格式说明
quatw x y z（4 个值）注意：MuJoCo 使用 wxyz 顺序，Isaac Gym 使用 xyzw 顺序，两者不同！
axisanglex y z angle（4 个值）旋转轴 + 旋转角度（弧度）
xyaxes6 个值分别指定局部 X 轴和 Y 轴在世界中的方向
zaxis3 个值指定局部 Z 轴在世界中的方向
eulerx y z（3 个值）欧拉角（弧度），旋转顺序为 XYZ

⚠️ MuJoCo vs Isaac Gym 的四元数顺序：

MuJoCo 使用 wxyz（w 在前），Isaac Gym 使用 xyzw（w 在末尾）。如果在这两个平台之间传递姿态数据，务必进行转换。

🏛 默认属性（default class）

通过 <default> 定义可复用的属性模版，避免重复编写：

<default>
    <!-- 全局 geom 默认：软接触参数 & 流体系数 -->
    <geom solref=".5e-4" solimp="0.9 0.99 1e-4"
          fluidcoef="0.5 0.25 0.5 2.0 1.0" />

    <!-- "card" class：扑克牌几何体 -->
    <default class="card">
        <geom type="mesh" mesh="card" mass="1.84e-4"
              contype="0" conaffinity="0" />
    </default>

    <!-- "collision" class：碰撞几何体（质量0，可视化用） -->
    <default class="collision">
        <geom type="box" mass="0" size="0.047 0.032 .00035"
              group="3" friction=".1" />
    </default>
</default>

9 本章小结
概念核心要点
geom有质量、参与碰撞的物理几何体；8 种类型，size 含义因类型而异
sitegeom 的轻量版，无质量无碰撞；仅用于可视化标记和传感器安装
body组织和嵌套物理实体的容器；父子关系形成坐标变换树
